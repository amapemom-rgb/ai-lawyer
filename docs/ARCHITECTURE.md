# Архитектура AI Lawyer v0.2

## Стек технологий

| Слой | Технология | Роль |
|------|-----------|------|
| Оркестрация | **LangGraph** | Граф состояний агента: маршрутизация запросов, условные переходы, повторные попытки |
| RAG-движок | **LlamaIndex** | Иерархическое разбиение текста, семантический поиск, субвопросная декомпозиция |
| Генерация | **Claude API** (через LangChain) | Формирование юридических ответов со ссылками на источники |
| Векторное хранилище | **ChromaDB** | Хранение эмбеддингов документов |
| Эмбеддинги | **multilingual-e5-base** | Локальная мультиязычная модель (не нужен API-ключ) |
| Граф знаний | **Neo4j** | Понятийно-графовая система (ПГС) — связи между законами, понятиями, прецедентами |
| API | **FastAPI** | REST-интерфейс |
| Интерфейс | **aiogram 3** | Telegram-бот |

## Граф агента (LangGraph)

```
входящий_запрос
    → почемучка (нужны ли уточнения?)
        → [да] → возврат уточняющих вопросов → END
        → [нет] → ретривал (LlamaIndex + ПГС)
            → генерация (Claude API)
                → валидация (проверка на галлюцинации)
                    → [не прошёл, попытка < 2] → повторная генерация
                    → [прошёл] → обновление ПГС → финальный ответ → END
```

## Потоки данных

### Запрос пользователя
1. Пользователь отправляет вопрос (Telegram / API)
2. **Почемучка** проверяет: достаточно ли контекста? Однозначен ли запрос?
3. **LlamaIndex Retriever** находит релевантные чанки в ChromaDB
4. **ПГС (Neo4j)** дополняет контекст связями между документами
5. **Claude** генерирует ответ на основе найденного контекста
6. **Валидатор** проверяет ответ: есть ли ссылки на источники, нет ли галлюцинаций
7. **ПГС** обновляется: новый прецедент взаимодействия

### Загрузка документа
1. Документ приходит (файл или API)
2. Парсинг (PDF, DOCX, TXT)
3. Классификация (закон, договор, инструкция, ...)
4. **LlamaIndex** разбивает на чанки (SentenceSplitter) и индексирует в ChromaDB
5. **ПГС** получает новый узел и связи с существующими документами

## Почему эта связка

### LlamaIndex для RAG (вместо голого ChromaDB)
- **SentenceSplitter** — чанки на границах предложений, не рвёт мысль посередине
- **Автоматические эмбеддинги** — не надо писать свою обвязку
- **Retriever API** — единый интерфейс для любого бэкенда (ChromaDB, Pinecone, Weaviate...)
- **Расширяемость** — SubQuestionQueryEngine, AutoMergingRetriever добавляются одной строкой

### LangGraph для оркестрации (вместо if/else)
- **Явный граф** — видно всю логику принятия решений
- **Условные переходы** — retry при провале валидации
- **Расширяемость** — добавить узел «проверка юрисдикции» = добавить одну функцию + одно ребро
- **Стандарт индустрии** — LangGraph де-факто стандарт для агентов в 2026

### Локальные эмбеддинги (multilingual-e5-base)
- Не нужен API-ключ для эмбеддингов
- Хорошо работает с русским языком
- Работает полностью локально — данные не уходят наружу
