# Архитектура AI Lawyer

## Обзор

Система состоит из пяти основных компонентов:

### 1. ИИ-Агент (сменяемый)
- **Файлы:** `src/agent/`
- **Суть:** Stateless-агент на базе Claude API. Всё состояние хранится в ПГС и базах.
- **Ротация:** Когда контекстное окно забивается, агент заменяется новым с чистым контекстом. Знания не теряются.

### 2. Модуль «Почемучка»
- **Файл:** `src/agent/pochemuchka.py`
- **Суть:** Если информации недостаточно — система задаёт вопросы, а не гадает. Каждый ответ пользователя обогащает ПГС.

### 3. Понятийно-Графовая Система (ПГС)
- **Файлы:** `src/pgs/`
- **Технология:** Neo4j
- **Суть:** Сердце системы. Хранит связи между понятиями, документами, прецедентами.
- **"Пузыри":** Когда тема набирает популярность, вокруг неё кристаллизуется отдельная экосистема.

### 4. База знаний
- **Файлы:** `src/knowledge/`
- **Технология:** ChromaDB (векторное хранилище)
- **Суть:** Семантический поиск по документам. Законы, договоры, правила, переписка.

### 5. Входной конвейер
- **Файлы:** `src/pipeline/`
- **Суть:** Парсинг → классификация → «почемучка» (если непонятно) → загрузка в базу знаний + ПГС.

## Потоки данных

### Запрос пользователя
```
Пользователь → Агент → ПГС (поиск связей) → База знаний (документы) → Агент (генерация) → Валидатор → Ответ
                                                                        ↓
                                                                  ПГС (обновление связей)
```

### Загрузка документа
```
Документ → Парсер → Классификатор → [Почемучка?] → База знаний + ПГС
```

### Ротация агента
```
Агент₁ (контекст полон) → Агент₂ (чистый контекст)
         ↓                          ↓
    ПГС + БЗ ←―――――――――――――――――――→ ПГС + БЗ  (одни и те же)
```

## Принципы

1. **Агент — расходный материал, знание — вечное**
2. **Лучше спросить, чем ошибиться**
3. **Каждое взаимодействие обогащает граф**
4. **Популярные темы кристаллизуются в пузыри**
