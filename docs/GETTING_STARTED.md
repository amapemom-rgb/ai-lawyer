# Быстрый старт — ИИ-Юрист

## Шаг 0: Быстрая проверка (без ключей, без установки зависимостей)

Хотите просто попробовать? Запустите автономное демо — оно работает на чистом Python без каких-либо зависимостей:

```bash
git clone https://github.com/amapemom-rgb/ai-lawyer.git
cd ai-lawyer
python scripts/demo_local.py
```

Это демо использует TF-IDF поиск по базе знаний. Вы увидите 5 тестовых запросов и сможете задавать свои вопросы. Никаких API-ключей не нужно.

---

## Шаг 1: Получить API-ключи

Для полной версии (с генерацией ответов через Claude) нужны ключи:

### 1.1 Anthropic API Key (для Claude)

1. Перейдите на [console.anthropic.com](https://console.anthropic.com)
2. Зарегистрируйтесь (или войдите)
3. Перейдите в **Settings → API Keys**
4. Нажмите **Create Key**
5. Скопируйте ключ (начинается с `sk-ant-...`)
6. Пополните баланс в **Billing** (для старта достаточно $5–10)

Стоимость: Claude Sonnet — примерно $3 на миллион входных токенов. Для тестирования — копейки.

### 1.2 Telegram Bot Token (опционально)

Если хотите Telegram-бота:

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Введите имя бота (например: `ИИ Юрист`)
4. Введите username (например: `ai_lawyer_test_bot`) — должен заканчиваться на `bot`
5. BotFather даст токен вида `7123456789:AAH...`

Это бесплатно.

## Шаг 2: Настроить проект

```bash
# Клонировать репозиторий (если ещё не сделали)
git clone https://github.com/amapemom-rgb/ai-lawyer.git
cd ai-lawyer

# Создать виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или: venv\Scripts\activate  # Windows

# Установить зависимости (первый раз скачает модель эмбеддингов ~1 ГБ)
pip install -r requirements.txt

# Создать файл настроек
cp .env.example .env
```

Откройте `.env` и заполните ключи:
```
ANTHROPIC_API_KEY=sk-ant-...ваш-ключ...
TELEGRAM_BOT_TOKEN=7123456789:AAH...ваш-токен...
```

## Шаг 3: Запуск

### Вариант А: CLI-демо (рекомендуется для первого теста)

```bash
python scripts/demo_cli.py
```

Загрузит базу знаний в память и запустит интерактивный режим. Задавайте юридические вопросы — система найдёт нужные документы и сгенерирует ответ через Claude.

Без API-ключа: покажет только результаты поиска (без генерации).
С API-ключом: полный RAG — поиск + ответ со ссылками.

### Вариант Б: Telegram-бот

```bash
python -m src.bot.telegram
```

Откройте своего бота в Telegram и напишите ему вопрос. Бот также принимает документы (PDF, DOCX, TXT) — загружает их в базу знаний.

### Вариант В: REST API

```bash
python -m src.api.main
```

API доступно на `http://localhost:8080`. Эндпоинты:
- `GET /health` — статус системы
- `POST /query` — запрос к ИИ-юристу (`{"question": "ваш вопрос"}`)
- `POST /documents/search` — поиск по базе знаний
- `POST /documents/upload` — загрузка документа

## Шаг 4: Тестирование

Пройдите тестовый сценарий из [docs/TESTING.md](TESTING.md) — 18 вопросов, разделённых на 4 уровня сложности.

Быстрая проверка — задайте боту:

1. «Сколько дней есть на возврат товара, купленного через интернет?»
2. «Кто отвечает за бракованный товар — маркетплейс или продавец?»
3. «Как отправить претензию на Ozon?»

Если бот отвечает со ссылками на конкретные статьи — Фаза 1 работает.

## Что дальше?

3. Добавьте ещё документы в базу знаний (можно скинуть PDF боту!)
4. Заполните итоговую таблицу в [docs/TESTING.md](TESTING.md)
5. Переходите к Фазе 2 (см. [docs/ROADMAP.md](ROADMAP.md))
